12679: Fix MAME Debugger GUI issues: horizontal disassembly scrolling and maxwidth [dave-br]
@tlindner : FYI
This fixes https://github.com/mamedev/mame/issues/12678
### PROBLEMS FIXED
1. Horizontally scrolling disassembly pane causes garbage to appear.
2. The parent debugger window refuses to be widened or horizontally scrolled even when disassembly contents are very wide (e.g., due
  to comments).
NOTE: I have observed the above problems on the MAME Windows Debugger under Windows.  I briefly tried the QT Debugger under Windows:
  Problem 1 occurs, but Problem 2 seems ok.  I briefly tried the imgui debugger, but it crashes every time I click on it.  I do not
  have access to MacOS or Linux.
### SCENARIOS FIXED
Anyone using the disassembly window (either embedded in the main window or free-floating) might hit problem (1) if the window is
  sized narrowly enough.  But the real issue is for folks (like me) who extensively use the comments feature.  In particular, I use
  a CoCo-based tool which allows the original assembly source you're debugging to appear as a comment for each machine code
  instruction (complete with labels and symbols).  This can make the information in the disassembly view quite wide (each comment
  alone could be about 100 characters).  Inability to widen the window or scroll the right portion of the comments into view is a
  pain.
### LIST OF CODE CHANGES
- debug_view_disasm::redraw: Fixed calls to print that were passing a "relative" end parameter (a column number based on where the
  horizontal scrollbar was situated) so they instead pass the expected "absolute" end parameter (a column number independent of what
  is horizontally scrolled into view).  This fixes the garbage (problem 1).
- debug_view_disasm: Replace hard-coded value of 50 for comment length with a dynamically calculated value based on the comments
  retrieved for the current buffer of disassembly to be shown (debug_view_disasm::complete_information).
- consolewin_info: Factor code that computes minwidth/maxwidth based on child windows so it may also be called from
  consolewin_info::recompute_children.
- debugwin_info::window_proc: Added call to recompute_children() at top of case WM_GETMINMAXINFO.  This ensures that when the system
  queries the parent window's min/max width, the parent first consults its children (specifically the disassembly window) so it
  calculates the values properly.
- Some minor changes to replace use of "int" in neighboring code with s32/u32.
